# Product

## Register

product

## Users

เจ้าหน้าที่สำนักงาน ก.ค.ศ. (สำนักงานคณะกรรมการข้าราชการครูและบุคลากรทางการศึกษา) และผู้บริหารระดับต้น–กลาง ใช้ระบบนี้ในสำนักงาน บนเครื่อง desktop เป็นหลัก ในบริบทราชการไทย ผู้ใช้มีความคุ้นเคยกับแบบฟอร์มกระดาษมากกว่าซอฟต์แวร์สมัยใหม่ ต้องการระบบที่ตรงไปตรงมา ใช้งานง่ายโดยไม่ต้องศึกษาเพิ่มเติม

## Product Purpose

ระบบประเมินค่างาน (JEVAL) ใช้สำหรับประเมินตำแหน่งบุคลากรทางการศึกษาตามมาตรา 38 ค.(2) ทั้งระดับปฏิบัติการ ระดับชำนาญการ และระดับชำนาญการพิเศษ ครอบคลุมกระบวนการตั้งแต่กรอกข้อมูลตำแหน่ง ตอบคำถามประเมิน ไปจนถึงแสดงผลและออกรายงาน PDF — ทดแทนกระบวนการกระดาษที่ล่าช้า ลดข้อผิดพลาด และให้ผลการประเมินที่โปร่งใสตรวจสอบได้

## Brand Personality

สะอาด (Clean) · ชัดเจน (Clear) · มีประสิทธิภาพ (Efficient)

น้ำเสียงเป็นทางการแบบราชการไทยสมัยใหม่ — สุภาพ ตรงประเด็น ไม่ใช้ศัพท์เทคนิค ไม่ informal เกินไป ให้ความรู้สึกว่าระบบนี้เชื่อถือได้และถูกต้องตามระเบียบ

## Anti-references

- เว็บราชการแบบเก่าที่ใช้ยาก: layout ไม่ responsive, ฟอนต์เล็ก, สีจืด, ฟอร์มยาวเกินไปในหน้าเดียว
- SaaS startup ต่างประเทศ: dashboard แน่นเกินไปด้วย widget, metric card, gradient สีฉูดฉาด — ไม่เข้ากับบริบทราชการไทย
- ซับซ้อนเกินไป: ระบบที่มีเมนู/ฟีเจอร์มากจนผู้ใช้ที่ไม่ชำนาญเทคโนโลยีสับสน

## Design Principles

1. **ชัดเจนตั้งแต่แรกเห็น** — ผู้ใช้ต้องรู้ทันทีว่าอยู่ขั้นตอนไหน ต้องทำอะไรต่อ โดยไม่ต้องเดา
2. **ลดภาระทางปัญญา** — แบ่งขั้นตอนเป็นส่วนย่อย แต่ละหน้าจอมีจุดประสงค์เดียว ไม่ถามหลายอย่างพร้อมกัน
3. **ถูกต้องตามระเบียบ** — ทุกขั้นตอน ทุกคำศัพท์ ต้องตรงกับกฎระเบียบ ก.ค.ศ. สร้างความเชื่อมั่นว่าผลลัพธ์ถูกต้อง
4. **เข้าถึงได้ทุกคน** — รองรับผู้ใช้ที่ไม่คุ้นเคยเทคโนโลยี ขนาดตัวอักษรอ่านง่าย ปุ่มกดได้ชัด ไม่พึ่งพาสีอย่างเดียวในการสื่อสาร
5. **ให้คุณค่าทันที** — จากข้อมูลที่กรอกถึงผลการประเมิน ควรเร็วและไม่มีขั้นตอนที่ไม่จำเป็น

## Accessibility & Inclusion

- เป้าหมาย WCAG 2.1 AA เป็นอย่างน้อย
- ข้อความภาษาไทยเป็นหลัก ใช้ฟอนต์ Noto Sans Thai — ต้องอ่านง่ายทุกขนาดหน้าจอ
- Contrast ratio ≥ 4.5:1 สำหรับ body text ทุกจุด
- รองรับ reduced motion: ผู้ใช้ราชการหลายคนอยู่ในวัยกลางคน–สูงอายุ
- ไม่พึ่งพาสีเพียงอย่างเดียวเพื่อสื่อสารสถานะ (สำเร็จ/ผิดพลาด) — ใช้ icon และ text ร่วมด้วยเสมอ
